### 实现资源包下载页面的计划

#### 1. 修改 ModrinthService.SearchModsAsync 方法
- 添加 `projectType` 参数，默认值为 "mod"
- 更新基础 facet 为 `project_type:{projectType}`
- 确保现有功能不受影响

#### 2. 更新 ResourceDownloadViewModel
- 添加资源包相关属性：
  - ResourcePackSearchQuery (搜索关键词)
  - ResourcePacks (资源包集合)
  - IsResourcePackLoading (加载状态)
  - IsResourcePackLoadingMore (加载更多状态)
  - ResourcePackOffset (偏移量)
  - ResourcePackHasMoreResults (是否有更多结果)
  - SelectedResourcePackVersion (选中的版本)
- 添加命令：
  - SearchResourcePacksCommand (搜索资源包)
  - LoadMoreResourcePacksCommand (加载更多资源包)
- 添加方法：
  - SearchResourcePacksAsync (搜索资源包的实现)
  - LoadMoreResourcePacksAsync (加载更多资源包的实现)

#### 3. 更新 ResourceDownloadPage.xaml
- 替换资源包占位符为完整的 UI 结构，参考 Mod 标签页
- 添加搜索框、版本筛选、排序等控件
- 添加资源包列表，使用 VirtualizingStackPanel 优化性能
- 添加加载指示器和错误信息显示

#### 4. 更新 ResourceDownloadPage.xaml.cs
- 添加事件处理程序：
  - ResourcePackSearchBox_QuerySubmitted (搜索提交)
  - ResourcePackVersionComboBox_SelectionChanged (版本筛选)
  - ResourcePackListScrollViewer_ScrollChanged (滚动加载更多)
  - ResourcePackItem_Tapped (资源包项点击)
- 更新 ResourceTabView_SelectionChanged 处理程序，支持资源包标签页的延迟加载

#### 5. 确保 API 调用正确
- 使用 `project_type:resourcepack` 作为 facet 参数
- 调用 Modrinth API 获取资源包数据
- 实现分页和加载更多功能

#### 6. 保持 UI 一致性
- 与 Mod 标签页使用相同的布局和样式
- 确保颜色、间距、字体等设计元素一致
- 实现相同的交互逻辑

#### 7. 测试和优化
- 测试资源包搜索、筛选、加载更多功能
- 优化性能，确保流畅的用户体验
- 修复可能的 bug

### 预期效果
- 资源包下载页面与 Mod 页面具有一致的 UI 设计和功能
- 支持搜索、版本筛选、排序等功能
- 实现分页加载，提高性能
- 正确调用 Modrinth API 获取资源包数据
- 支持延迟加载，提高 TabView 切换流畅度
