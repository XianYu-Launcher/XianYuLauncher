# 实现图标下载并使用的逻辑

## 1. 需求分析
- 在下载Mod、资源包等项目时，额外下载一份图标至本地
- 在使用时优先使用本地图标
- 确保图标在不同屏幕尺寸下正确显示

## 2. 实现方案

### 2.1 图标存储位置设计
- 在Minecraft数据目录下创建统一的`icons`文件夹
- 按照项目类型（mod/resourcepack/shader/datapack/map）进行子目录分类
- 图标命名规则：`{项目ID}_{版本ID}.png` 或 `{文件名}_icon.png`

### 2.2 下载逻辑修改
1. **修改`DownloadModAsync`方法**：
   - 添加图标下载逻辑
   - 在下载主文件后，检查并下载图标
   - 保存图标到指定位置

2. **添加图标下载辅助方法**：
   - `DownloadIconAsync`：负责下载图标到本地
   - 支持进度更新和错误处理

3. **修改ModVersionViewModel**：
   - 添加`IconUrl`属性，存储图标URL
   - 确保在创建ModVersionViewModel时包含图标信息

### 2.3 版本管理页面修改
1. **修改ModInfo类**：
   - 添加`IconPath`属性，存储本地图标路径

2. **修改版本管理ViewModel**：
   - 在加载Mod/资源包时，检查本地是否存在图标
   - 优先使用本地图标，否则使用默认图标

3. **修改列表项模板**：
   - 更新图标绑定，支持本地图标路径

### 2.4 图标显示优化
- 使用`BitmapImage`加载本地图标
- 设置适当的`DecodePixelWidth`和`DecodePixelHeight`，确保图标清晰且不占用过多内存
- 处理图标加载失败的情况，显示默认图标

## 4. 代码实现

### 4.1 ModDownloadDetailViewModel.cs修改
```csharp
// 添加图标下载逻辑到DownloadModAsync方法
private async Task DownloadIconAsync(string iconUrl, string savePath)
{
    try
    {
        using (HttpClient client = new HttpClient())
        {
            var iconBytes = await client.GetByteArrayAsync(iconUrl);
            await File.WriteAllBytesAsync(savePath, iconBytes);
        }
    }
    catch (Exception ex)
    {
        // 图标下载失败不影响主文件下载
        System.Diagnostics.Debug.WriteLine("图标下载失败: " + ex.Message);
    }
}

// 在DownloadModAsync方法中调用
// 下载主文件后，添加图标下载
if (!string.IsNullOrEmpty(ModIconUrl) && !ModIconUrl.StartsWith("ms-appx:"))
{
    string iconDir = Path.Combine(minecraftPath, "icons", ProjectType);
    _fileService.CreateDirectory(iconDir);
    string iconPath = Path.Combine(iconDir, $"{ModId}_{Path.GetFileNameWithoutExtension(modVersion.FileName)}_icon.png");
    await DownloadIconAsync(ModIconUrl, iconPath);
}
```
