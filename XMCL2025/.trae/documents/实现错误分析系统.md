# 错误分析系统实现计划

## 1. 系统设计目标

* 收集完整的错误信息

* 显示详细的错误分析弹窗

* 实现日志导出和打包功能

* 预留错误解决方法接口

## 2. 核心功能实现

### 2.1 错误信息收集

* 修改 `ReadProcessOutputAsync` 方法，将所有输出保存到内存中

* 创建 `_gameOutput` 和 `_gameError` 字符串列表，实时记录游戏输出

* 在进程退出时保存完整日志

### 2.2 详细错误弹窗设计

* 创建新的 `ErrorAnalysisDialog` 类，继承自 `ContentDialog`

* 弹窗包含以下部分：

  * 错误标题和时间

  * 完整的错误信息显示

  * 错误分析区域（预留）

  * 操作按钮：

    * 确定：关闭弹窗

    * 导出日志：触发日志打包功能

    * 查看详情：展开/折叠完整日志

### 2.3 日志导出功能

* 创建 `ExportCrashLogsAsync` 方法

* 实现以下功能：

  * 生成 `启动参数.bat` 文件，包含完整的启动命令

  * 生成 `输出日志.txt` 文件，包含所有游戏输出

  * 使用 `System.IO.Compression` 打包为 `minecraft_crash_<时间>.zip`

  * 保存到用户指定位置（默认桌面）

### 2.4 错误检测和触发

* 修改错误检测逻辑，不再对每一行ERROR显示弹窗

* 改为在进程异常退出时显示完整的错误分析弹窗

* 保留实时ERROR日志记录

## 3. 代码修改点

### 3.1 启动ViewModel.cs

* 添加 `_gameOutput` 和 `_gameError` 字段

* 修改 `ReadProcessOutputAsync` 方法，保存所有输出

* 修改 `MonitorGameProcessExitAsync` 方法，在异常退出时显示错误分析弹窗

* 添加 `ExportCrashLogsAsync` 方法

* 添加 `ShowErrorAnalysisDialog` 方法

### 3.2 新增文件

* `Views/ErrorAnalysisDialog.xaml`：错误分析弹窗UI

* `Views/ErrorAnalysisDialog.xaml.cs`：错误分析弹窗逻辑

## 4. 技术实现要点

### 4.1 异步操作

* 所有文件操作使用异步方法

* 弹窗操作在UI线程上执行

### 4.2 日志格式

* 启动参数.bat：包含完整的Java命令

* 输出日志.txt：包含时间戳和完整的游戏输出

### 4.3 打包功能

* 使用 `ZipFile.CreateFromDirectory` 或 `ZipArchive` 实现打包

* 生成唯一的文件名，包含时间戳

### 4.4 用户体验

* 弹窗设计简洁明了

* 提供清晰的操作选项

* 导出过程显示进度

* 导出完成后显示成功提示

## 5. 预留扩展接口

* 添加 `ErrorSolutionProvider` 类，预留错误解决方法接口

* 设计错误类型枚举，便于后续扩展

* 预留错误数据库接口，用于存储和查询错误解决方法

## 6. 实现步骤

1. 修改 `ReadProcessOutputAsync` 方法，保存所有输出
2. 修改 `MonitorGameProcessExitAsync` 方法，添加错误分析弹窗触发
3. 创建 `ErrorAnalysisDialog` 弹窗
4. 实现 `ExportCrashLogsAsync` 方法
5. 测试完整功能

## 7. 预期效果

* 游戏异常退出时，显示详细的错误分析弹窗

* 弹窗包含完整的错误信息和操作选项

* 点击"导出日志"可生成包含启动参数和完整日志的zip文件

* 系统稳定可靠，不影响游戏正常运行

* 预留扩展接口，便于后续添加错误解决方法功能
